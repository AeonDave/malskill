# x86-64 Reference

## Register Table

| Register | 32-bit | 16-bit | 8-bit | Role |
|---|---|---|---|---|
| `rax` | `eax` | `ax` | `al` | Return value, accumulator |
| `rbx` | `ebx` | `bx` | `bl` | Callee-saved general purpose |
| `rcx` | `ecx` | `cx` | `cl` | Arg 4, loop counter, shift count |
| `rdx` | `edx` | `dx` | `dl` | Arg 3, high half of mul/div |
| `rsi` | `esi` | `si` | `sil` | Arg 2, string source |
| `rdi` | `edi` | `di` | `dil` | Arg 1, string destination |
| `rbp` | `ebp` | `bp` | `bpl` | Frame pointer (callee-saved) |
| `rsp` | `esp` | `sp` | `spl` | Stack pointer |
| `r8`–`r11` | `r8d`–`r11d` | `r8w`–`r11w` | `r8b`–`r11b` | Args 5–6 (`r8`/`r9`), caller-saved |
| `r12`–`r15` | `r12d`–`r15d` | `r12w`–`r15w` | `r12b`–`r15b` | Callee-saved |

**SIMD**: `xmm0`–`xmm7` caller-saved (args/return); `xmm8`–`xmm15` callee-saved on Windows, caller-saved on Linux.

## Win64 ABI Details

### Shadow Space Layout

After `call`, the stack looks like:

```
[rsp+0x28] arg5 (if >4 args)
[rsp+0x20] shadow slot 4 (home for r9)
[rsp+0x18] shadow slot 3 (home for r8)
[rsp+0x10] shadow slot 2 (home for rdx)
[rsp+0x08] shadow slot 1 (home for rcx)
[rsp]      return address
```

Callee may spill register args into shadow space. Caller always allocates 32 bytes even for 0 args.

### Register Preservation (System V vs Win64)

| Register | System V | Win64 |
|---|---|---|
| `rdi`, `rsi` | Caller-saved (args 1-2) | **Callee-saved** |
| `xmm6`–`xmm15` | Caller-saved | **Callee-saved** (must save/restore 128-bit) |
| `rcx` | Arg 4 | Arg 1 |
| Red zone | 128 bytes below RSP | **None** |

### Win64 Exception Handling

- Every non-leaf function needs a `RUNTIME_FUNCTION` entry (begin, end, unwind info)
- Prolog operations: push reg, sub rsp, mov [rsp+N] — each mapped to `UNWIND_CODE`
- RSP must not change between prolog end and epilog start
- Use `.pdata` and `.xdata` sections (PE/COFF) or `/SAFESEH` linker flag
- Frame pointer optional; RSP-relative addressing is standard

## Instruction Selection Guide

### Arithmetic

| Goal | Preferred | Avoid |
|---|---|---|
| `x * 2` | `lea rax, [rdi*2]` or `shl rdi, 1` | `imul rdi, 2` |
| `x * 3` | `lea rax, [rdi + rdi*2]` | `imul rdi, 3` |
| `x * 9` | `lea rax, [rdi + rdi*8]` | `imul rdi, 9` |
| `x / 2` (signed) | `sar rdi, 1` | `idiv` |
| `x / 2` (unsigned) | `shr rdi, 1` | `div` |
| `x % power-of-2` | `and rdi, (N-1)` | `div` |
| `abs(x)` | `mov rax, rdi` / `neg rax` / `cmovl rax, rdi` | branch |

### Conditional / Branchless

```nasm
; if (a < b) result = a else result = b  →  min(a, b)
cmp     rdi, rsi
cmovg   rdi, rsi        ; rdi = min(rdi, rsi)
mov     rax, rdi
ret

; bool x = (a == b)
xor     eax, eax        ; zero-extend to rax — faster than xor rax,rax
cmp     rdi, rsi
sete    al
ret
```

### Memory Access and Alignment

- **16-byte alignment** required for `movaps`, `movdqa` — use `movups`/`movdqu` for unaligned
- **32-byte alignment** required for `vmovaps` / `vmovdqa` (AVX)
- **64-byte alignment** for AVX-512

```nasm
; Align a label/data to 16 bytes
align 16
.hot_loop:
    ; loop body
```

### Zero / Sign Extension

```nasm
movzx   eax, byte [rdi]     ; zero-extend byte → 32-bit (also clears upper 32)
movzx   eax, word [rdi]     ; zero-extend word → 32-bit
movsx   rax, dword [rdi]    ; sign-extend 32-bit → 64-bit
cdqe                        ; sign-extend eax → rax
```

## SSE2 / AVX2 Patterns

### Horizontal Sum (4 floats → 1)

```nasm
; xmm0 = [a, b, c, d] → xmm0[0] = a+b+c+d
haddps  xmm0, xmm0          ; [a+b, c+d, a+b, c+d]
haddps  xmm0, xmm0          ; [a+b+c+d, ...]
```

### Byte-Level Search (find zero byte)

```nasm
; Check 16 bytes for zero byte using PCMPEQB + PMOVMSKB
movdqu  xmm0, [rdi]
pxor    xmm1, xmm1
pcmpeqb xmm0, xmm1          ; 0xFF per byte == 0
pmovmskb eax, xmm0           ; bitmask: bit N set if byte N was 0
test    eax, eax
jnz     .found_zero
```

### AVX2 — 32-byte integer compare

```nasm
vmovdqu  ymm0, [rdi]
vpcmpeqb ymm0, ymm0, ymm1   ; ymm1 = value to find
vpmovmskb eax, ymm0
```

## AVX-512 Patterns

### Opmask Registers (k0–k7)

```nasm
; k-registers control per-element masking
vpcmpd   k1, zmm0, zmm1, 1  ; k1 = mask where zmm0 < zmm1
vpaddd   zmm2{k1}{z}, zmm0, zmm1  ; add only where k1 set; zero others
```

### 64-byte Aligned Load + Process

```nasm
align 64
.loop:
    vmovdqa64  zmm0, [rdi + rcx]     ; 64-byte aligned load
    vpaddd     zmm0, zmm0, zmm1      ; add bias
    vmovdqa64  [rsi + rcx], zmm0     ; store result
    add        rcx, 64
    cmp        rcx, rdx
    jb         .loop
    vzeroupper                        ; not needed for 512-bit, but safe habit
```

## Cryptographic & CRC Instructions

### AES-NI (single round)

```nasm
; AES-128 encryption: 10 rounds
pxor      xmm0, xmm1             ; initial AddRoundKey
aesenc    xmm0, xmm2             ; round 1
aesenc    xmm0, xmm3             ; round 2
; ... rounds 3-9 ...
aesenclast xmm0, xmm11           ; final round
```

**Feature check**: `CPUID.(EAX=01h):ECX.AES[bit 25] = 1`

### CRC32 (hardware)

```nasm
; CRC32C (Castagnoli polynomial) — SSE4.2
crc32   eax, byte [rdi]          ; accumulate 1 byte
crc32   eax, dword [rdi]         ; accumulate 4 bytes
crc32   rax, qword [rdi]         ; accumulate 8 bytes (64-bit operand)
```

### SHA Extensions (Intel SHA-NI)

```nasm
; SHA-256 — 2 rounds at a time
sha256rnds2  xmm1, xmm2          ; implicit xmm0 = message schedule
sha256msg1   xmm3, xmm4          ; message schedule expansion
sha256msg2   xmm3, xmm5
```

## Atomic Operations

### Compare-and-Swap Loop

```nasm
; atomic CAS: [rdi] = new if [rdi] was expected
; input: rsi=expected, rdx=new
atomic_cas:
    mov     rax, rsi             ; rax = expected (cmpxchg compares with rax)
.retry:
    lock cmpxchg [rdi], rdx      ; if [rdi]==rax: [rdi]=rdx, ZF=1
    jnz     .retry               ; failed: rax now holds current value
    ret                          ; success: rax = old expected value
```

### Spinlock

```nasm
; Simple test-and-test-and-set spinlock
acquire:
.spin:
    pause                        ; reduce power on spin-wait
    cmp     dword [rdi], 0       ; test without bus lock
    jne     .spin
    mov     eax, 1
    xchg    [rdi], eax           ; atomic exchange (implicit LOCK)
    test    eax, eax
    jnz     .spin                ; someone else got it
    ret

release:
    mov     dword [rdi], 0       ; plain store = release (x86 TSO)
    ret
```

### Fence Usage Guide

| When | Fence |
|---|---|
| After non-temporal stores (`movnti`, `vmovntps`) | `sfence` |
| Before `rdtsc` for accurate timing | `lfence` (serializes dispatch) |
| Sequential consistency needed | `mfence` or use `lock` instruction |
| Normal loads/stores | None needed (x86 TSO already ordered) |

## Performance Quick-Reference

### Instruction Cost Categories

| Latency | Examples |
|---|---|
| 1 cycle | `mov`, `add`, `sub`, `and/or/xor`, `lea`, `cmp`, `test` |
| 3 cycles | `imul r, r`, `cmov`, `shl/shr` (variable count) |
| 3–5 cycles | `mulps/mulpd`, `addps/addpd` (SIMD float) |
| 5 cycles | `vfmadd*` (FMA) |
| 20–35+ cycles | `div`/`idiv` (avoid in hot loops) |
| 4+ cycles | Memory load (L1 hit); 12 (L2); 40+ (L3); 200+ (DRAM) |

### Common Performance Pitfalls

- **Dependency chains**: break long serial chains; use multiple accumulators for reductions
- **Store-forwarding stalls**: writing then reading same address with different widths; keep read/write widths matching
- **Branch misprediction**: ~15 cycles penalty; use `cmov` for simple conditionals
- **Partial register stalls**: `mov al, X` followed by `mov rbx, rax` — use `movzx eax, X` instead
- **SSE/AVX transition penalty**: call `vzeroupper` after AVX code before calling non-AVX code
- **Misaligned SIMD**: `movaps`/`vmovaps` on unaligned data = #GP fault; use `movups`/`vmovups`
- **`div` in loops**: replace with reciprocal multiply + shift when dividing by constant

### Alignment Guidelines

| Target | Min Alignment |
|---|---|
| Hot loop entry | 16 bytes (or 32 for best I-cache) |
| Jump targets in hot code | 16 bytes |
| Stack at `call` | 16 bytes |
| Data for SSE | 16 bytes |
| Data for AVX | 32 bytes |
| Data for AVX-512 | 64 bytes |
| Lock variables | Cache-line (64 bytes) to avoid false sharing |

## ABI Edge Cases

- **varargs**: `al` must hold the number of used XMM registers before calling a variadic function
- **`syscall` clobbers**: `rcx`, `r11` (used internally); use `r10` instead of `rcx` for arg 4
- **Red zone**: 128 bytes below `rsp` are safe in leaf functions (System V only, NOT Win64)
- **TLS** (`fs` / `gs` segment): do not modify segment registers; `fs:0x28` = stack canary on Linux
- **Return address**: always `[rsp]` at function entry; push/pop disturbs this
- **Stack alignment on Win64**: no red zone; RSP must be 16-byte aligned at `call` but callee receives misaligned RSP (return address pushed)
